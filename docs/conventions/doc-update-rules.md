## CHANGELOG.md

- 기존 동작 변경 시 `Changed` 항목 추가
- 버그 수정 시 `Fixed` 항목 추가
- 기능/API 제거 시 `Removed` 항목 추가
- 배포 전 버전 태그(날짜 or 버전) 확정

## CLAUDE.md
- 새로운 코딩 패턴이 프로젝트에 도입될 때
- 금지 패턴이 생길 때
- 디렉토리 구조가 바뀔 때
- 팀 규칙 또는 컨벤션이 변경될 때

## README.md
- 새로운 기능이 추가되어 소개가 필요할 때
- 실행 방법 또는 환경 설정이 바뀔 때
- 외부 의존성(API, 서비스)이 추가될 때

## docs/planning/requirements.md
- 기획 요구사항이 추가되거나 변경될 때
- 클라이언트/이해관계자와 합의된 내용이 바뀔 때
- 기능 범위(scope)가 조정될 때

## docs/planning/scenarios.md
- 유저 플로우가 추가되거나 변경될 때
- 예외 케이스(엣지 케이스)가 새로 정의될 때
- 화면 흐름이 재설계될 때

## docs/planning/decisions.md
- 라이브러리 또는 프레임워크를 선택할 때
- 아키텍처 구조를 결정할 때
- 대안을 검토하고 하나를 선택한 모든 순간
- 형식: 날짜 / 결정 내용 / 이유 / 고려했던 대안

## docs/planning/schema.md
- 테이블이 추가되거나 삭제될 때
- 컬럼이 추가, 변경, 삭제될 때
- 인덱스 또는 제약조건이 바뀔 때
- Flyway migration 파일 생성 시 동기화
